# Codebook — `consumer_data_raw.csv`

Synthetic teaching data for a consumer decision-making experiment. **300 participants**, **5 conditions**. Generated by [`_generate.R`](./_generate.R) with `set.seed(20260602)`.

## Design

Participants saw a product description framed by one of five **conditions** (a messaging manipulation) and then reported attitudes and a hypothetical purchase decision.

## Variables

| Raw name | Cleaned name | Type | Description |
| --- | --- | --- | --- |
| `ParticipantID` | `participant_id` | character | Unique participant code (`P001`–`P300`). |
| `Condition` | `condition` | factor (5) | `control`, `scarcity_frame`, `sustainability_frame`, `social_proof_frame`, `combined_frame`. **Has trailing whitespace in raw — needs `trimws()`.** |
| `Age` | `age` | integer | Participant age in years. **4 missing.** |
| `Gender` | `gender` | factor | Inconsistent labels in raw (`Woman`, `woman`, `W`, `Female`, …). **Needs recoding.** |
| `IncomeGroup` | `income_group` | ordered factor | `low` < `medium` < `high`. **`-99` codes 8 missing values.** |
| `ProductCategory` | `product_category` | factor | `electronics`, `food`, `clothing`, `services`. |
| `ps_1`, `ps_2`, `ps_3` | → `price_sensitivity` | int 1–7 | Three Likert items measuring price sensitivity. **Average to form scale.** |
| `sc_1`, `sc_2`, `sc_3` | → `sustainability_concern` | int 1–7 | Three Likert items. **Average.** |
| `bt_1`, `bt_2`, `bt_3_r` | → `brand_trust` | int 1–7 | Three items measuring brand trust. **`bt_3_r` is reverse-scored — reverse before averaging.** |
| `pv_1`, `pv_2`, `pv_3` | → `perceived_value` | int 1–7 | Three items. **Average. This is the mediator.** |
| `perceived_quality` | `perceived_quality` | int 1–7 | Single-item measure. |
| `perceived_social_norm` | `perceived_social_norm` | int 1–7 | Single-item measure. |
| `pi_1`, `pi_2`, `pi_3` | → `purchase_intention` | int 1–7 | Three items. **Average. This is the primary outcome.** |
| `willingness_to_pay` | `willingness_to_pay` | numeric | Euros the participant would pay (≥ 0). Secondary outcome. |
| `choice` | `choice` | factor (no/yes) | Binary buy decision (0 = no, 1 = yes). |
| `number_products_bought` | `number_products_bought` | integer 0–5 | Count outcome. |

## Baked-in effects

The data has real effects you can find:

1. **Main effect:** `condition` predicts `purchase_intention`; the `combined_frame` is the strongest (≈ +0.57 on a 7-point scale, *p* ≈ .007).
2. **Mediation:** `condition` → `perceived_value` → `purchase_intention`. Adding `perceived_value` to the outcome model substantially attenuates the condition coefficients.
3. **Moderation:** `price_sensitivity` moderates the condition effect — the boost from `combined_frame` is weaker among high-price-sensitive participants (interaction *p* ≈ .047).

## Realistic messiness (and why)

| Mess | Where | Why it's there |
| --- | --- | --- |
| Trailing whitespace | `Condition` | Common copy-paste artefact. Teaches `trimws()` or `stringr::str_squish()`. |
| Inconsistent labels | `Gender` | Real-world free-text or merged-survey messes. Teaches recoding. |
| `-99` missing codes | `IncomeGroup` | Standard SPSS convention. Teaches `na_if()`. |
| Reverse-scored item | `bt_3_r` | Real psychometric scales do this. Teaches `8 - x`. |
| Mixed case column names | All capitalised names | Teaches `janitor::clean_names()`. |
| Scattered `NA`s | Likert items, `age` | Teaches missing-value handling and `complete.cases()`. |
